Both compile to the same DOM — this very tour ships as showcase.html and showcase-md.html.

So which one should you use? Wrong question, friends — both compile to the same DOM, and you can mix them slide by slide in a single deck. But let's compare anyway.

⟨CLICK⟩

Surface: a plain section of HTML tags, versus a data-markdown section wrapping a CommonMark template. Same destination, different commute.

⟨CLICK⟩

Builds: one attribute in HTML, a triple-colon fence in Markdown.

⟨CLICK⟩

Notes: an aside element, or Note: and Rehearse: blocks. Either way, they land in the speaker view.

⟨CLICK⟩

Now the honest part: diagrams, auto-animate, fine layout — that's HTML's home turf. In Markdown you drop back to raw HTML exactly where the visuals get ambitious.

⟨CLICK⟩

So: HTML when you want control, Markdown when you're moving fast — or when the author is an agent. Which, around here, is often.
